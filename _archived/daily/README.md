## 深思熟虑？！

不能不明不白做需求，完成工作。

以前

- 一句话需求 → 先开发完成 → 验收时不合要求，再修改
- 用 PHP 等动态语言，也不涉及底层数据存储时，就是调用底层接口获取数据再操作

现在

- 用 Java 等静态语言，无论编译、部署、调试，反馈周期都长很多，试错成本高
- 还需要考虑底层的数据存储的存取，需要的时间多好几倍

正确方式

- 背景 → 需求 → 方案 → 实现 ( → 测试 → 上线 )
- 即使只是一句话「需求」，也得搞清楚「背景」
- 「方案」确定了，最后一步才是去「实现」即写代码
